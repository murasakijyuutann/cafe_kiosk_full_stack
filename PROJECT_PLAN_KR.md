# 카페 키오스크 프로젝트 계획서 - 4주 부트캠프

**프로젝트 기간:** 4주
**팀 규모:** 5명
**대상:** 초급 부트캠프 프로젝트
**제출 날짜:** 2025/11/11

---

## 요약

### 프로젝트 개요
고객이 메뉴를 탐색하고, 장바구니를 관리하며, 주문을 완료할 수 있는 셀프 서비스 카페 키오스크 웹 애플리케이션입니다. 이 풀스택 프로젝트는 백엔드에 Spring Boot를, 프론트엔드에 React를 사용하여 현대적인 웹 개발 기술을 실습할 수 있습니다.

### 학습 목표
- 처음부터 완전한 풀스택 애플리케이션 구축
- Git과 GitHub를 사용한 팀 협업 실습
- RESTful API 아키텍처 구현
- 관계형 데이터베이스 설계 작업 (MySQL)
- React로 반응형 프론트엔드 인터페이스 개발
- 애자일 개발 방법론 적용

### 프로젝트 범위
**포함 사항:**
- 카테고리별 메뉴 탐색
- 장바구니 관리
- 주문 생성 및 확인
- 주문 번호로 주문 조회

**제외 사항 (4주 내):**
- 사용자 인증/로그인
- 결제 처리
- 관리자 대시보드
- 실시간 알림

## 데이터베이스 설계

### ERD (개체-관계 다이어그램)

```
┌─────────────────┐
│    category     │
├─────────────────┤
│ id (PK)         │
│ name            │
│ description     │
│ display_order   │
│ created_at      │
└────────┬────────┘
         │
         │ 1:N
         │
┌────────▼────────┐
│   menu_item     │
├─────────────────┤
│ id (PK)         │
│ category_id(FK) │
│ name            │
│ description     │
│ price           │
│ image_url       │
│ available       │
│ created_at      │
│ updated_at      │
└────────┬────────┘
         │
         │ N:M (order_item을 통해)
         │
┌────────▼────────┐         ┌─────────────────┐
│   order_item    │──────▶ │     orders      │
├─────────────────┤  N:1  ├─────────────────┤
│ id (PK)         │       │ id (PK)         │
│ order_id (FK)   │       │ order_number    │
│ menu_item_id(FK)│       │ customer_name   │
│ quantity        │       │ total_amount    │
│ unit_price      │       │ status          │
│ subtotal        │       │ ordered_at      │
└─────────────────┘       └─────────────────┘
```

### 테이블 요약
- **categories** - 메뉴 항목 분류 (커피, 디저트, 음료)
- **menu_items** - 가격 및 세부 정보가 포함된 개별 제품
- **orders** - 고객 주문 기록
- **order_items** - 주문과 메뉴 항목을 수량과 함께 연결

- 사용자가 모든 메뉴 카테고리 볼 수 있음
- 사용자가 카테고리별로 메뉴 항목 필터링 가능
- 메뉴 항목에 이름, 설명, 가격, 이미지 표시
- 사용 가능한 항목만 표시됨

**FR-2: 장바구니**
- 사용자가 수량과 함께 장바구니에 항목 추가 가능
- 사용자가 장바구니 내용 볼 수 있음
- 사용자가 항목 수량 업데이트 가능
- 사용자가 장바구니에서 항목 제거 가능
- 장바구니에 소계 및 총 가격 표시
- 장바구니가 브라우저에 유지됨 (localStorage)

**FR-3: 주문 처리**
- 사용자가 장바구니를 주문으로 제출 가능
- 시스템이 고유한 주문 번호 생성
- 시스템이 총 금액 계산
- 사용자가 주문 확인 받음
- 선택적 고객 이름 필드

**FR-4: 주문 조회**
- 사용자가 주문 번호로 주문 세부 정보 볼 수 있음
- 주문 세부 정보에 모든 항목, 수량, 가격 포함

### 비기능 요구사항

**성능:**
- 페이지 로드 시간 < 3초
- API 응답 시간 < 500ms

**사용성:**
- 반응형 디자인 (태블릿 및 데스크톱에서 작동)
- 명확한 탐색
- 직관적인 사용자 인터페이스
- 유용한 오류 메시지

**코드 품질:**
- 일관된 코드 형식
- 의미 있는 변수/함수 이름
- 기본 오류 처리
- 복잡한 로직에 주석

## API 엔드포인트

### 메뉴 API
| 메서드 | 엔드포인트 | 설명 | 응답 |
|--------|----------|-------------|----------|
| GET | `/api/menu/categories` | 모든 카테고리 가져오기 | 카테고리 목록 |
| GET | `/api/menu/items` | 모든 메뉴 항목 가져오기 | 메뉴 항목 목록 |
| GET | `/api/menu/items/category/{id}` | 카테고리별 항목 가져오기 | 필터링된 메뉴 항목 |

### 주문 API
| 메서드 | 엔드포인트 | 설명 | 요청 본문 | 응답 |
|--------|----------|-------------|--------------|----------|
| POST | `/api/orders` | 새 주문 생성 | OrderRequest | 주문 번호가 포함된 OrderResponse |
| GET | `/api/orders/{orderNumber}` | 번호로 주문 가져오기 | - | 주문 세부 정보 |

**요청/응답 예시:**

**주문 생성 요청:**
```json
{
  "customerName": "홍길동",
  "items": [
    {
      "menuItemId": 1,
      "menuItemName": "아메리카노",
      "price": 3000,
      "quantity": 2,
      "subtotal": 6000
    }
  ]
}
```

**주문 생성 응답:**
```json
{
  "id": 1,
  "orderNumber": "ORD-20250111-001",
  "customerName": "홍길동",
  "totalAmount": 6000,
  "status": "PENDING",
  "orderedAt": "2025-01-11T14:30:00",
  "items": [...]
}
```

- 의미 있는 컴포넌트 이름 사용

## 테스트 전략

### 백엔드 테스트
- **수동 테스트:**
  - 각 서비스 메서드 테스트
  - 데이터베이스 변경 확인
  - Postman으로 API 엔드포인트 테스트

- **테스트 항목:**
  - 메뉴 항목이 올바르게 로드됨
  - 카테고리가 적절하게 필터링됨
  - 주문 생성이 데이터베이스에 저장됨
  - 주문 번호가 고유하게 생성됨

### 프론트엔드 테스트
- **수동 테스트:**
  - 모든 사용자 플로우 클릭
  - 다양한 화면 크기에서 테스트
  - 다양한 브라우저에서 테스트

- **테스트 시나리오:**
  - 메뉴 탐색 → 항목 추가 → 장바구니 보기 → 체크아웃
  - 각 카테고리별 필터링
  - 장바구니 수량 업데이트
  - 장바구니에서 항목 제거
  - 고객 이름 있음/없음으로 주문 제출
  - 빈 장바구니 동작

### 통합 테스트
- 전체 플로우 테스트: 프론트엔드 → API → 데이터베이스 → 응답
- 데이터 일관성 확인
- 오류 처리 확인

## 위험 관리

### 식별된 위험 및 완화

| 위험 | 확률 | 영향 | 완화 방법 |
|------|------------|--------|------------|

| 기술적 어려움 | 높음 | 중간 | 일일 체크인, 서로 돕기, 강사에게 질문 |

| 시간 부족 | 중간 | 높음 | 핵심 기능 우선순위화, 필요시 부가 기능 제외 |
